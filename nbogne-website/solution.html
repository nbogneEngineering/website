How nBogne Works

Your medical data shouldn't need phone calls and faxes to move between clinics. Here's how we make that possible — architecturally, not contractually.

The Insight

SWIFT for Healthcare

SWIFT moves trillions of dollars globally without ever holding the money. They route it, confirm delivery, log the transaction — but your money goes directly from your bank to theirs.

We borrowed that idea for medical records. nBogne routes data between systems, translates formats, and logs everything. But your clinical data goes straight from source to destination.

Trust doesn't come from contracts. It comes from architecture.

Core Capabilities

Format Transformation

Real-time conversion between healthcare data standards.

  • HL7v2 (2.3, 2.5, 2.7)
  • FHIR R4
  • JSON / REST APIs
  • CSV / Flat files
  • Custom schemas on request

Intelligent Compression

Critical for bandwidth-constrained environments.

  • 60-80% payload reduction
  • Z-Standard algorithm
  • Works on 2G/GPRS networks
  • Typical 5KB → 1.5KB

Guaranteed Delivery

Messages persist until successfully delivered.

  • SQLite queue (survives power loss)
  • Exponential backoff retry
  • Dead letter queue for inspection
  • Delivery confirmation

Metadata-Only Audit

Full compliance without storing patient data.

  • Transaction ID & timestamp
  • Source & destination
  • Cryptographic hash (integrity proof)
  • Never: names, IDs, clinical content

The Data Journey

What happens when EMR A needs to send a lab result to EMR B.

1

EMR Sends Data

Source EMR sends patient data via API. Data arrives in whatever format the EMR supports — HL7v2, FHIR, proprietary JSON.

2

nBogne Receives

Data loaded into RAM. Never touches disk. Clock starts on 60-second maximum processing window.

3

Format Transformation

Our format library identifies source format and transforms to destination's expected format automatically.

4

Compression & Transmission

Payload compressed 60-80%. Routing rules determine destination. Transmitted with retry logic.

5

Delivery Confirmation

Destination EMR confirms receipt. nBogne logs transaction metadata only: ID, timestamp, source, destination, size, status, hash.

6

Data Purge

Clinical data cleared from RAM. Typical time: under 10 seconds. Maximum allowed: 60 seconds. No trace remains.

Security Through Architecture

Traditional security asks: "How do we protect stored data?" We ask: "What if there's nothing to protect?"

No Data at Rest

Clinical data never touches disk. RAM-only processing means even physical server access yields nothing.

No Honeypot

Centralized health data repositories are high-value targets. nBogne has no repository to target.

Cryptographic Integrity

Every transaction includes a SHA-256 hash proving content wasn't altered — without revealing content.

Breach = Metadata Only

Worst case: transaction logs exposed. When data moved, between whom, how large. Never what was in it.

Vendor Exit Safety

If nBogne shuts down tomorrow, no patient data is affected. We can't lose what we don't have.

Compliance by Default

Ghana DPA, HIPAA, GDPR — architecture handles compliance. Nothing to erase because nothing is stored.
